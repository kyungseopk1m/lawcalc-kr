## TIER-A: v0.1.3

1. A1 - Fix period-mode `segment.days` vs formula numerator asymmetry.
   - Status: completed in v0.1.1 by redefining period-mode `segment.days` to match formula numerator days.
   - Owner: A.
2. A2 - Define and fix `addYears` clip semantics.
   - Decision: treat 02-29 to next-year 02-28 as the clipped anniversary for one-year display purposes, but keep the calculation policy explicit in tests and docs. The implementation must not silently change interest totals without golden fixture updates.
   - Owner: A.
3. A3 - Revisit `totalDays` denominator behavior for intervals longer than one year.
   - Choose either split-by-year calculation or a documented 1-year maximum for `totalDays` mode.
   - Owner: A.
4. A4 - Remove the `customRate` zero fallback in `App.tsx`.
   - Status: completed by B-W7 (`033ff58`) by returning `undefined` for invalid custom rates and preserving validation errors instead of silently using 5%.
   - Invalid custom rates should stay invalid, not silently become 5%.
   - Owner: B.
5. A5 - Validate `.lcalc` loaded input instead of filling missing fields with silent defaults.
   - Status: completed by B-W7 (`033ff58`) via `parseLoadedLcalcInput`.
   - Missing required fields should produce a user-facing toast and preserve the current form state.
   - Owner: B/C.
6. A6 - Add Rust CI gates.
   - Implemented in D-W10: `cargo fmt --all -- --check` and `cargo clippy --all-targets -- -D warnings`.
   - Owner: D.
7. A7 - Add a decimal custom-rate fixture.
   - Include a multi-segment rate such as `12.345%` to catch floating-point accumulation drift.
   - Owner: A.
