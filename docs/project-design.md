# Project Design

lawcalc-kr 는 한국 법률 계산을 로컬에서 검토할 수 있게 하는 데스크톱 워크벤치입니다. 이 문서는 공개 README보다 조금 더 구현 중심으로 도메인 경계와 릴리스 흐름을 정리합니다.

## 1. 제품 표면

현재 데스크톱 앱은 이자 / 상속 / 소송비용 / 변제충당 / 손해배상 5개 도메인을 탭으로 제공합니다. 각 도메인은 입력, 계산 결과, 데이터 버전, 저장·내보내기 흐름을 같은 UX 원칙으로 공유합니다.

## 2. 로컬 우선 원칙

사건 입력값은 사용자 기기 안에서만 처리합니다. `.lcalc` 파일, PDF, CSV, 클립보드 출력은 모두 로컬 액션이며 서버 전송을 요구하지 않습니다.

## 3. 엔진 경계

도메인 계산은 `packages/core-engine` 의 순수 TypeScript 함수가 담당합니다. React/Tauri 레이어는 폼 상태, 파일 IO, export, 네이티브 대화상자만 담당합니다.

## 4. 데이터 버전 관리

법령·실무·수학표 데이터는 `data/<dataset>/v<N>.json` 을 단일 출처로 두고, 동기화 스크립트가 생성된 TypeScript 데이터셋으로 변환합니다. 결과 객체와 `.lcalc` 저장 형식에는 사용한 데이터셋 식별자를 포함합니다.

## 5. `.lcalc` 기능 ID

저장 파일은 `schemaVersion: "3"` 저장 형식을 사용합니다. 기능 ID는 `interest@1`, `inheritance@1`, `litigation-cost@1`, `appropriation@1`, `compensation@1`처럼 도메인별로 명시합니다.

## 6. 면책 고지 표면

면책 고지는 `packages/core-engine/src/disclaimers.ts` 의 `STANDARD_DISCLAIMER` 를 단일 출처로 사용합니다. 신규 출력 표면은 이 문자열을 직접 복제하지 않고 결과나 내보내기 흐름을 통해 전달합니다.

## 7. 공개 산출물 정책

공개 저장소에는 공식 매뉴얼 본문, 설치 파일, 디컴파일 산출물, 사적인 동기 설명을 포함하지 않습니다. 공개 문서는 출처명, URL, 독립적으로 작성한 설명, 계산 입력·기대 출력만 보존합니다.

## 8. 손해배상 로드맵

v0.5.0 은 손해배상 도메인의 첫 공개 범위로 `compensation@1` 을 도입합니다. 범위는 자동차 사고 부상이며, 기초사항, 노동력상실률, 일실수입, 위자료, 과실상계, 공제, 호프만 240 한도, 데이터셋 경과 안내까지 포함합니다.

| 릴리스 | 범위            | 기능 ID          | 비고                                                                                                                                                                             |
| ------ | --------------- | ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| v0.5.0 | 자×부상 첫 범위 | `compensation@1` | 대한건설협회 시중노임, KOSIS 생명표, 호프만, 라이프니츠 데이터셋 4종을 결과에 동봉합니다.                                                                                        |
| v0.6.0 | 자×사망 (완료)  | `compensation@2` | 생계비 1/3 공제, 장례비, 상속지분 분배를 추가했습니다. 상속 분배는 상속분 간이 계산을 재사용하므로 1991-01-01 이후 사망 케이스를 대상으로 하며 이를 release notes 에 명시합니다. |
| v0.7.0 | 산재 (완료)     | `compensation@3` | 사건종류(자동차/산재)를 직교 축으로 추가해 산재 부상은 장해급여, 산재 사망은 유족급여를 과실상계 후 공제 단계에 더합니다. 자동차 결과는 회귀 없이 그대로 유지합니다.             |
| v0.8.0 | 기타손해        | `compensation@4` | 개호비, 치료비, 보조구 등 별도 손해 항목을 추가합니다.                                                                                                                           |

손해배상 도메인은 v0.5.2 에서 `@lawcalc-kr/compensation` 과 `@lawcalc-kr/datasets-compensation` 두 패키지로 분리했습니다. 자×사망(`compensation@2`)은 같은 패키지의 `auto-death/` 하위에 구현하고, 상속 분배는 `@lawcalc-kr/core-engine` 의 상속 엔진을 재사용합니다. 산재(`compensation@3`)는 별도 엔진을 두지 않고 부상·사망 입력에 사건종류(`accidentType`) 축과 산재보험급여 공제 필드만 더해, 자동차 엔진을 그대로 재사용합니다.
